Security Coverage

These protections directly target the main weaknesses from the prior vulnerable prototype.

SQL Injection: raw string-built SQL has been replaced with parameterized queries.

XSS: templated output remains escaped and user input is constrained before storage.

Broken Authentication: passwords are hashed and repeated login failures are rate-limited.

Input Validation: usernames, roles, IDs, marks, and upload file types are checked before processing.

Access Control: admin-only actions enforce role checks both in the UI flow and at the route level.

Session Security: cookies are HTTP-only, same-site, time-bounded, and protected by security headers.
